Example deployment-config.yaml (conceptual)

A simplified, illustrative configuration showing how controls and environments might be declared. This is example-only, not a template. 

environment: production
region: eu-west-2

controls:
  logging: centralised
  encryption_at_rest: true
  encryption_in_transit: true
  zero_trust:
    enabled: true
    identity_provider: "organisation-idp"

workloads:
  - name: payments-api
    runtime: container
    network_segment: "payments"
    data_classification: "confidential"
    compliance:
      - "SOC 2"
      - "PCI-DSS"
  - name: analytics-platform
    runtime: managed-service
    network_segment: "analytics"
    data_classification: "restricted"
    compliance:
      - "SOC 2"
